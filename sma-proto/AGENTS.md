# SMA-Proto Module Guide

**Location**: `sma-proto/`
**Domain**: gRPC Protocol Buffer definitions for cross-language RPC
**Language**: Protocol Buffers (proto3)
**Score**: 8/25 (interface definitions, generates Go/Rust code)

## Overview

Universal language-agnostic interface definitions (IDL) for high-performance RPC between Go Orchestration Layer and Rust Physical Execution Layer. Uses protobuf for efficient binary serialization.

## Structure

```
sma-proto/
├── sandbox.proto      # SandboxManager service definition
├── go.mod            # Go module for generated code
└── README.md         # Usage documentation
```

## Conventions (This Module)

### Proto Structure
```protobuf
syntax = "proto3";
package sma;
option go_package = "sma-os/sma-proto/sandbox";
```

### Field Numbering
```protobuf
message AssignRequest {
  string tenant_id = 1;    // Sequential from 1
  string namespace = 2;
  string action_name = 3;
  string payload_json = 4;
}
```

### Service Definition
```protobuf
service SandboxManager {
  rpc AssignMicroVM (AssignRequest) returns (AssignResponse);
  rpc TeardownMicroVM (TeardownRequest) returns (TeardownResponse);
}
```

## Anti-Patterns (This Module)

### Forbidden
```protobuf
// NEVER: Skip field numbers (breaks compatibility)
string tenant_id = 1;
string namespace = 3;  // WRONG: Skipped 2

// ALWAYS: Sequential numbering
string tenant_id = 1;
string namespace = 2;  // CORRECT
```

### Breaking Changes
```protobuf
// WRONG: Changing field types (breaks compatibility)
// Was: string vm_id = 1;
int32 vm_id = 1;  // BREAKING CHANGE

// CORRECT: Add new field, deprecate old
string vm_id = 1 [deprecated = true];
string vm_id_v2 = 5;
```

## Commands

```bash
# Generate Go code
protoc --go_out=. --go_opt=paths=source_relative \
  --go-grpc_out=. --go-grpc_opt=paths=source_relative \
  sandbox.proto

# Generate Rust code (using tonic-build)
# In Cargo.toml build-dependencies:
# tonic-build = "0.10"
```
